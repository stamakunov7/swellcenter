## Introduction: Business Need and Problem

The purpose of this project was to design and build a simple e-business web application prototype called **The Swell Center**. The concept is a wellness center that combines fitness support, mental wellness support, recovery services, and basic appointment booking into one website. The business need behind this application is that many people want to improve their health, but they often have to manage different providers and systems for each type of service. This can make wellness feel complicated and hard to maintain.

The Swell Center prototype addresses this problem by presenting a single, organized online location where users can learn about available services, compare options, and submit a booking request. For a college assignment, this project demonstrates practical web design principles, basic e-business functionality, and user-centered communication. The solution is intentionally beginner-friendly and focuses on clarity over technical complexity.

## Design Approach and Key Features

The application was designed as a multi-page static website with three core pages: a home page, a services page, and a booking page. The home page introduces the center and explains what problem it solves. It also includes a visible call-to-action that directs users to the booking page, which supports the business goal of converting visitors into potential clients.

The services page presents offerings in a structured table with service name, duration, price, and availability. This format helps users quickly compare options and understand value. The booking page includes a clean form that asks for core appointment details, such as name, contact information, preferred service, and preferred schedule. Form fields are grouped with a fieldset and legend to improve readability and accessibility.

From a visual perspective, the design uses a calm color palette, clear spacing, rounded cards, and readable typography to reflect the wellness brand identity. A shared external stylesheet keeps styling consistent and easier to maintain. The overall user experience is meant to feel professional while still being simple enough for a student to explain step-by-step during a presentation.

## Implementation Tools, Platforms, and Languages

This prototype was created using:

- **HTML5** for semantic page structure.
- **CSS3** for layout, visual design, and responsive behavior.
- **GitHub Codespaces** as the development environment.
- **GitHub repository structure** prepared for deployment to **GitHub Pages**.

No framework was used, and JavaScript was intentionally avoided to keep the project aligned with assignment expectations and beginner-level maintainability. Because the app is static, it is straightforward to host and publish, and it avoids backend complexity.

## Accessibility and Usability Features

Accessibility was considered during structure and styling decisions. Semantic HTML elements (header, nav, main, section, article, footer) improve document meaning for screen readers and assistive technologies. Navigation is consistent across all pages and includes clear text labels. A clickable image includes descriptive alternative text for non-visual users.

On the booking page, all form controls include associated labels, which supports keyboard and screen-reader navigation. The form also groups related fields using fieldset and legend for better context. The services table uses proper headings and row/column structure to remain understandable when read by assistive tools.

In terms of responsive design, CSS media queries adjust layout for larger screens while preserving mobile usability. This helps ensure that users can access the same content effectively from phones, tablets, and laptops.

## Development Challenges and Resolutions

One challenge in the project was balancing a professional appearance with beginner-level code readability. It can be tempting to add advanced design effects, but doing so would make the code harder to explain in class. This was resolved by using simple reusable components (cards, buttons, and form layout) and clear comments in files.

Another challenge was creating a design that looked modern without using frameworks such as Bootstrap. This was handled by writing a custom stylesheet with variables for colors, spacing, and consistent visual structure. A third challenge was making the table and form both accessible and visually clean; this was addressed by using semantic markup first and then layering CSS styling on top.

## Future Improvements

If this prototype were expanded into a production e-business application, several improvements could be added:

1. Real backend booking storage and confirmation emails.
2. User accounts for saved appointments.
3. Admin dashboard for managing service schedules.
4. Payment processing for prepaid sessions.
5. More wellness content such as blog posts, FAQ, and testimonials.
6. Stronger analytics integration for conversion tracking.

These improvements would move the project from prototype stage to a more complete digital business platform.
